## 二、交互流程说明

### 2.1 漏洞管理核心流程

1. **漏洞上报**：用户填写表单 → 上传附件 → 提交 → 自动生成ID → 通知负责人
2. **漏洞分派**：PM查看待分派列表 → 选择处理人 → 分派 → 状态变更
3. **漏洞修复**：开发人员接收任务 → 修复 → 标记已修复 → 等待复核
4. **漏洞复核**：测试人员验证 → 通过则关闭 → 不通过则重新打开

### 2.2 扫描集成流程

1. 创建扫描任务 → 配置目标 → 启动扫描
2. 扫描完成 → 自动导入结果 → 生成漏洞报告
3. 去重检测 → 避免重复漏洞
